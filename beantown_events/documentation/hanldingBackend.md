* Using vercel hosted server to hanbdle requests, and a separate js file to house the bacckend logic, we ensure the backend is not directly incldued or exposed in the front end by: 
    1. including "use server" at top of backend file. This ensures that when the files are compiled by vercel server, this file gets striped out of front end deployment and reserved in the server as a function reachable via https requests (POST, GET, etc). This comes with a catch: since the backend functions are not incldued in the front end deployment, they are actually only referenced by the front end, with a unique hash address/ID (random string), when triggerred. Often this makes the function a ghost function on the front end. Then, the front end would send a request to the backend - te backedn would read the hash id and will invoke that function tied to that hash id. Each funciton in the bakcend gets assigned a hash id.
